# 自动化安全评估系统使用说明书

## 1. 系统定位

本系统用于对已授权资产执行被动式安全评估，适合做基线检查、元数据采集和合规性验证。系统不会执行漏洞利用、爆破、绕过或持久化行为。

## 2. 运行前准备

### 2.1 环境要求

- Python 3.11 或更高版本
- Windows PowerShell、命令提示符或兼容终端
- 本地可写目录：项目根目录下的 `data/` 与 `reports/`

### 2.2 初始化数据库

```bash
python main.py init-db
```

执行后会在 `data/autopentest.db` 创建本地数据库。

## 3. 典型操作流程

### 3.1 创建授权任务

```bash
python main.py create-engagement ^
  --name "内网资产基线检查" ^
  --description "办公网核心系统被动检查" ^
  --authorized-by "安全负责人" ^
  --ticket "SEC-2026-001" ^
  --owner "蓝队" ^
  --allow-host "app.internal.example.com" ^
  --allow-prefix "https://portal.internal.example.com"
```

说明：

- `--allow-host` 用于指定允许访问的主机名或通配域名
- `--allow-prefix` 用于指定允许访问的 URL 前缀
- 至少需要提供一项 `--allow-host` 或 `--allow-prefix`

### 3.2 查看任务列表

```bash
python main.py list-engagements
```

### 3.3 添加目标

```bash
python main.py add-target --engagement-id 1 --url "https://app.internal.example.com" --label "门户首页"
```

如果目标不在授权范围内，系统会直接拒绝。

### 3.4 查看目标列表

```bash
python main.py list-targets --engagement-id 1
```

### 3.5 运行被动检查作业

```bash
python main.py run-job --engagement-id 1 --requested-by "ops.user"
```

当前作业会执行以下检查：

- 安全响应头检查
- Cookie 安全属性检查
- TLS 证书与协议检查

### 3.6 导出报告

导出 Markdown：

```bash
python main.py report --job-id 1 --format md
```

导出 JSON：

```bash
python main.py report --job-id 1 --format json
```

如需自定义输出位置：

```bash
python main.py report --job-id 1 --format md --output ".\\reports\\custom-job-1.md"
```

## 4. Web 控制台使用

### 4.1 启动服务

默认端口已调整为 `8081`：

```bash
python main.py serve
```

等价于：

```bash
python main.py serve --host 127.0.0.1 --port 8081
```

启动后访问：

`http://127.0.0.1:8081`

### 4.2 页面功能说明

- 顶部状态区：显示 API 状态、当前选中的任务和最近一次作业
- 创建任务：录入任务名称、工单号、授权人、负责人和范围
- 已保存任务：切换当前任务
- 目标资产：为当前任务添加或查看目标
- 被动检查作业：运行当前任务的检查作业并查看发现
- 报告预览：直接查看 Markdown 报告内容

## 5. REST API 概览

### 5.1 健康检查

- `GET /api/health`

### 5.2 任务相关

- `GET /api/engagements`
- `POST /api/engagements`
- `GET /api/engagements/{id}`

### 5.3 目标相关

- `GET /api/engagements/{id}/targets`
- `POST /api/engagements/{id}/targets`

### 5.4 作业与报告

- `POST /api/engagements/{id}/jobs`
- `GET /api/jobs/{id}`
- `GET /api/jobs/{id}/report?format=md`
- `GET /api/jobs/{id}/report?format=json`

## 6. 常见问题

### 6.1 无法添加目标

常见原因：

- URL 不在授权范围内
- URL 缺少协议或主机名
- URL 中包含账号密码

### 6.2 页面打不开

请确认：

- `python main.py serve` 已启动成功
- 本机 `8081` 端口未被占用
- 浏览器访问的是 `http://127.0.0.1:8081`

### 6.3 报告为空

请确认：

- 当前任务下已存在目标
- 已执行过至少一次作业
- 目标站点在执行窗口内可访问

## 7. 安全边界

- 仅允许对已授权范围内目标执行检查
- 仅收集 HTTP/TLS 元数据
- 不包含攻击性利用行为
- 不适用于未授权目标
